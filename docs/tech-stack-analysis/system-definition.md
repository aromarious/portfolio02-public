# システム本質定義書

## 🎯 システムの本質

### 何のシステムか

**ポートフォリオサイト（個人開発者向け）**

- 主目的: 技術力アピール・採用活動での差別化
- 副目的: 実際の問い合わせ受付・顧客獲得

### 運用目的・期間

以下の短期目標のみを考える。

- **短期目標**: 転職活動での技術力証明（6ヶ月-1年）
- **中期目標**: フリーランス・案件獲得（1-3年）
- **長期目標**: 個人ブランディング・技術者としての信頼構築（3年+）

### システム規模・特性

- **ユーザー数**: 月間数百〜数千PV（採用担当者・クライアント候補）
- **問い合わせ**: 月間0-10件程度
- **運用体制**: 個人開発者1名のみ
- **予算**: 個人レベル（月額数千円以内）

## 🎯 技術選択の判断基準

### ✅ 必要な要素（技術力アピール重視）

1. **モダン技術スタック**: Next.js 15, React 19, TypeScript, tRPC v11
2. **企業レベル品質**: テスト、型安全性、エラーハンドリング
3. **セキュリティ意識**: 基本的なセキュリティ対策実装
4. **運用自動化**: CI/CD、監視、バックアップの基本実装
5. **パフォーマンス**: 高速レスポンス、UX最適化

### ❌ 不要な要素（個人開発には過剰）

1. **大規模組織向け機能**: 多段階承認、チーム管理、アクセス制御
2. **エンタープライズ向け規制**: SOX法、GDPR、PCI DSS、HIPAA
3. **大規模インフラ**: 複数リージョン、シャーディング、カオスエンジニアリング
4. **24/7運用体制**: オンコール、インシデント対応チーム
5. **高コスト機能**: 有料監視ツール、専用サーバー、高可用性構成

### 🔄 条件付き必要（規模・要件による）

1. **Google Analytics**: 実際の効果測定が必要な場合
2. **外部API連携**: 実際の業務効率化が必要な場合
3. **詳細ログ**: 問題発生時の調査が必要な場合
4. **バックアップ**: データ保護が重要な場合

## 🎯 評価基準の統一

### 評価軸

1. **技術力アピール度**: 採用担当者への印象効果
2. **実装コスト**: 開発時間・学習コスト
3. **運用コスト**: 月額費用・メンテナンス負荷
4. **実用性**: 実際の問い合わせ処理への貢献度

### 判定基準

- **○ (必要)**: 技術力アピール効果が高く、個人開発レベルで実用的
- **△ (条件付き)**: 特定条件下でのみ必要、または技術検証として有用
- **× (不要)**: 個人開発には過剰、コスト・複雑性に見合わない

## 🎯 これからのレビュー方針

この定義書を基準として、ID:101以降の項目を以下の観点で一括判定：

1. **ポートフォリオサイトとしての必要性**
2. **個人開発者の技術力アピール効果**
3. **実装・運用コストの妥当性**
4. **採用面接での差別化効果**

この基準により、1項目ずつの詳細検討ではなく、カテゴリー単位での効率的な判定が可能になります。
