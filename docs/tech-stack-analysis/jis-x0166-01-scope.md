# JIS X 0166:2014 第1章 適用範囲

## 1 適用範囲

この規格の適用範囲を次に示す。

- システム・ソフトウェア製品（サービスを含め）に対する要求事項を工学的に扱うためにライフサイクル各段階において実施することが必要なプロセスを規定する。

- JIS X 0160:2012及びJIS X 0170:2013に記述されている要求事項を扱う諸プロセス（以下，要求プロセスという。注記1参照）及び要求事項に関連する諸プロセス（以下，要求関連プロセスという。注記1参照）を適用するための指針を与える。

> **注記1** 要求プロセスとは，利害関係者要求事項定義プロセス（JIS X 0160:2012及びJIS X 0170:2013），要求事項分析プロセス（JIS X 0170:2013），及びソフトウェア要求事項分析プロセス（JIS X 0160:2012）を指す。要求関連プロセスとは，要求に関連するアクティビティ・タスクをもつ技術プロセス及びプロジェクトプロセスで，方式設計，検証，妥当性確認，構成管理，情報管理及び測定プロセスである（6.1及び附属書Cを参照）。

- 要求プロセスの実施を通じて作成することが要求される情報項目を規定する。

- 情報項目に要求される内容を規定する。

- 要求される情報項目及び関連情報項目の様式に対する指針を与える。

この国際規格は，以下に適用する。

- JIS X 0160:2012及びJIS X 0170:2013を使用する又は使用を計画している人

- 要求エンジニアリング活動を実行しようとする人

- 人が作るシステム，ソフトウェア中心システム，ソフトウェア・ハードウェア製品を取り扱うプロジェクトの関係者

この国際規格は，顧客，取得者，供給者，開発者，運用者，保守要員，経営陣，品質保証要員及び利用者によって使用される。

利用者は，プロセス規定若しくは情報項目規定，又はその両方への適合性を主張することができる。

この国際規格では，4 段階の適合性を考慮している（6.3参照）。

> **注記2** この国際規格は，ISO/IEC 15288 及び ISO/IEC 12207 で定義されているプロセス名称を使用している。しかし，この国際規格は，ISO/IEC 15288 及び ISO/IEC 12207 のライフサイクルプロセスに適合するプロジェクトに限定されるものではない。
