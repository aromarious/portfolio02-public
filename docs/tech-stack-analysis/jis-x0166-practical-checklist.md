# JIS X 0166準拠 実用的システム要求チェックリスト

## 概要

このドキュメントは、JIS X 0166:2014「システム及びソフトウェア技術－ライフサイクルプロセス－要求エンジニアリング」に基づいて、実際のシステム構築プロジェクトで使用可能な実用的チェックリストを提供します。

## JIS X 0166準拠 品質チェック

### 個々の要求事項の品質特性（9項目）

各要求項目について、以下の特性を満たしているかチェック：

1. **必要である** ✓/×

   - この要求は本当に必要か？
   - ビジネス価値を提供するか？

2. **実装独立である** ✓/×

   - 具体的な実装方法に依存していないか？
   - 「何を」であって「どのように」ではないか？

3. **曖昧性がない** ✓/×

   - 解釈が複数存在しないか？
   - 明確で理解しやすいか？

4. **一貫性がある** ✓/×

   - 他の要求と矛盾していないか？
   - 用語の使用に一貫性があるか？

5. **完全である** ✓/×

   - 必要な情報が全て含まれているか？
   - 前提条件や制約が明記されているか？

6. **単独である** ✓/×

   - 1つの要求は1つの事項のみを扱うか？
   - 複数の要求が混在していないか？

7. **実現可能である** ✓/×

   - 技術的に実現可能か？
   - 予算・時間制約内で実現できるか？

8. **追跡可能である** ✓/×

   - 上位要求から追跡できるか？
   - 実装・テストと関連付けられるか？

9. **検証可能である** ✓/×
   - 要求が満たされたことを確認できるか？
   - 客観的な判定基準があるか？

### 要求事項集合の品質特性（4項目）

要求全体について、以下の特性を満たしているかチェック：

1. **完全である** ✓/×

   - すべての必要な要求が含まれているか？
   - TBD/TBS/TBR項目がないか？

2. **一貫性がある** ✓/×

   - 要求間で矛盾がないか？
   - 用語・記法に一貫性があるか？

3. **満足可能である** ✓/×

   - すべての要求を同時に満たすことができるか？
   - 相互に排他的な要求がないか？

4. **定義範囲が明確である** ✓/×
   - システムの境界が明確か？
   - 対象範囲・除外範囲が明確か？

## 実装状況評価

### 評価スケール

- ✅ **完全実装** (100%): 要求を完全に満たし、品質特性も満たしている
- 🟡 **部分実装** (50-99%): 要求の一部を満たしているが、改善余地がある
- ❌ **未実装** (0-49%): 要求を満たしていない、または大幅な改善が必要
- 📋 **要検討** (N/A): 要件が不明確、またはプロジェクト範囲外

### 総合評価指標

- **JIS X 0166準拠率** = (完全実装項目数 ÷ 総項目数) × 100%
- **品質特性満足率** = (品質特性満足項目数 ÷ 総チェック項目数) × 100%

## 活用方法

1. **プロジェクト開始時**: 要求定義のベースライン確立
2. **設計フェーズ**: アーキテクチャ設計の妥当性確認
3. **実装フェーズ**: 実装漏れ・品質問題の早期発見
4. **レビュー時**: 要求品質の客観的評価
5. **監査対応**: 標準準拠の証跡提供

## まとめ

このチェックリストは、JIS X 0166の理論を実用的なツールに変換したものです。システム構築プロジェクトにおいて：

- **要求の網羅性確保**: 見落としがちな運用要件も含めて包括的にカバー
- **品質の客観的評価**: 標準に基づく明確な評価基準
- **継続的改善**: プロジェクト全体を通じた品質向上

日本の産業標準に準拠しながら、実際の開発現場で使える実用性を重視した内容となっています。
